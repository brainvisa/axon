BrainVISA 6.0

     
       

Website : https://brainvisa.info

New in 6.0:

  • The Morphologist pipeline has been added a summary report process step, and some global morphometry which may be used as QC with normative values.
  • A new snapshot utility process is available, which is a simpler version of the former "SnapBase" tool, which had been removed in 5.1 versions because its maintenance was discontinued. The new process is named "snapbase", as a replacement.
  • The BIDS files organization for Morphologist has been widely changed, in order to fit more standard BIDS schemas, and fix problems in the earlier version. For this reason, data processed with the former (1.0) BIDS schema will not be reusable with the newer one (2.0).

New in 5.1:

  • Support for BIDS-like data organization: databases can now use ontologies morphologist-bids-1.0 or freesurfer-bids to describe or organize processed data in a BIDS-like manner. raw-bids-1.0 can be used for input raw data as input for import processes.
  • A few other small fixes

See The changelog.

Quick start

  • Create a database where all data written by BrainVISA will be stored : BrainVISA menu -> Preferences -> Databases -> Add button. It is strongly advisable to use a database to process data with BrainVISA. Indeed, some important features are not available when you are using data outside a database. More information about databases here.
  • Import raw data : Data Management toolbox -> import... Choose the process according to the type of data. Select raw data with . Fill in database attributes using . The process will copy data in BrainVISA database.
  • Process data : Open a process, enter input parameters by selecting them in the database using . BrainVISA automatically complete as many parameters as possible. Output data will be written in the database.
